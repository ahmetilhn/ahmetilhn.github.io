# Proje Kod Kültürü ve Kuralları

Bu dosya, projenin kod kültürünü, mimari kararlarını ve kurallarını tanımlar.
Yeni projeler oluştururken bu dosyayı AI'ya vererek aynı kod standartlarını koruyabilirsin.

---

## 1. Teknoloji Yığını

- **Kütüphane:** React
- **Dil:** TypeScript (strict mode)
- **Bundler:** Vite
- **Build Çıktısı:** UMD (widget olarak embed edilebilir)
- **State Management:** Zustand (tek dosyada store)
- **Styling:** SCSS Modules (component-scoped) + Global SCSS (variables, mixins, utility classes)
- **HTTP Client:** Axios (custom interceptors ile)
- **Linting:** ESLint + Prettier
- **Formatting:** Prettier
- **Git Hooks:** Husky (commit öncesi lint)
- **Icons:** Custom SVG React component'leri
- **CSS Class Merge:** clsx

## 3. Widget Mimarisi

Proje, başka bir uygulama içerisine embed edilebilen bir widget olarak tasarlanmıştır.

### Entry Point (`src/widget.tsx`)
- Class tabanlı API: `new ExampleWidget(params).init()`
- `init()` ile mount, `destroy()` ile unmount
- Parametre olarak `targetElemId`, `isLoggedIn`, `onClose` alır

### Build
- `npm run dev` → Vite dev server (`index.html` + `src/main.tsx`)
- `npm run build` → UMD bundle (`build/example-widget.umd.js` + `build/*.css`)
- React ve ReactDOM external olarak beklenir (host proje sağlar)

### Kullanım
```html
<div id="chat-widget"></div>
<script>
  const widget = new ExampleWidget({
    targetElemId: 'chat-widget',
    isLoggedIn: false,
    onClose: () => console.log('kapatıldı'),
  });
  widget.init();
</script>
```

## 4. Bileşen Kuralları

### Yapı
- Her component kendi klasöründe `index.tsx` + opsiyonel `index.module.scss` dosyası ile bulunur.
- Tüm component'lar **functional component** olarak yazılır.
- Component'ler `FC<Props>` ile tiplendirilir.
- Performance için `memo()` ile sarılır.

### Örnek Bileşen Yapısı

```tsx
import { FC, memo } from 'react';
import clsx from 'clsx';

import UIVariant from '@/enums/ui-variant.enum';

import styles from './index.module.scss';

type Props = {
  title: string;
  variant?: UIVariant;
  isActive?: boolean;
};

const MyComponent: FC<Props> = ({ title, variant = UIVariant.PRIMARY, isActive }) => {
  return (
    <div className={clsx(styles.wrapper, isActive && styles['wrapper--active'])}>
      <h2>{title}</h2>
    </div>
  );
};

export default memo(MyComponent) as typeof MyComponent;
```

### Props Tanımlama
- `type` kullanılır (`interface` değil).
- Opsiyonel prop'lar `?` ile işaretlenir.
- HTML attribute'leri extend etmek için `Omit` ve `Partial` kullanılır.
- Ortak prop'lar `CommonElementProps` type'ından türetilir.

```tsx
type Props = Omit<ButtonHTMLAttributes<HTMLButtonElement>, 'onClick'> &
  Partial<{
    variant: UIVariant;
    size: UISize;
    isLoading: boolean;
  }> &
  CommonElementProps;
```

## 6. Import Kuralları

- Tüm import'lar `@/*` path alias'ı ile yapılır (relative import kullanılmaz).
- `tsconfig.json` ve `vite.config.ts` içinde `"@/*": ["src/*"]` tanımlıdır.

### Import Sıralaması

```tsx
// 1. External kütüphaneler
import { FC, memo, useState } from 'react';
import clsx from 'clsx';

// 2. Internal - component'ler
import Button from '@/components/shared/Button';
import Modal from '@/components/shared/Modal';

// 3. Internal - hook'lar
import useChat from '@/hooks/useChat';

// 4. Internal - servisler
import chatService from '@/services/modules/chat.service';

// 5. Internal - store
import { useChatStore } from '@/store';

// 6. Internal - type'lar ve enum'lar
import ChatStatus from '@/enums/chat-status.enum';
import type { ChatMessage } from '@/types/chat-message.type';

// 7. Internal - helper'lar ve constant'lar
import { formatDate } from '@/helpers/date.helper';

// 8. Styles (her zaman en sonda)
import styles from './index.module.scss';
```

### Export Kuralları
- Component'ler: `export default memo(ComponentName) as typeof ComponentName;`
- Utility fonksiyonlar: Named export
- Store: Named export (`export const useChatStore = create<ChatStore>(...)`)
- Servisler: Default export (object olarak)

## 7. Durum Yönetimi (Zustand)

### Store Yapısı
- Tek bir store dosyası: `store/index.ts`
- `create()` ile store oluşturulur.
- State ve action'lar aynı objede tanımlanır.

```tsx
import { create } from 'zustand';

import ChatStatus from '@/enums/chat-status.enum';
import type { ChatMessage } from '@/types/chat-message.type';

type ChatStore = {
  messages: ChatMessage[];
  status: ChatStatus;

  addMessage: (message: ChatMessage) => void;
  setStatus: (status: ChatStatus) => void;
  resetChat: () => void;
};

export const useChatStore = create<ChatStore>(set => ({
  messages: [],
  status: ChatStatus.IDLE,

  addMessage: message => set(state => ({ messages: [...state.messages, message] })),
  setStatus: status => set({ status }),
  resetChat: () => set({ messages: [], status: ChatStatus.IDLE }),
}));
```

### Store Kullanımı
- Component içinde: `const { messages, addMessage } = useChatStore();`
- React dışında (widget class): `useChatStore.getState().resetChat()`

## 8. Stil Kuralları (SCSS Modülleri)

### Genel Kurallar
- Her component kendi `index.module.scss` dosyasına sahiptir.
- Global değişkenler `_variables.scss`'de tanımlanır ve `vite.config.ts` içindeki `css.preprocessorOptions.scss.additionalData` ile tüm SCSS dosyalarına otomatik import edilir.
- BEM (Block Element Modifier) naming convention'ı kullanılır.
- Responsive tasarım mixin'ler ile yapılır (`@include for-mobile`, `@include for-tablet`).

### SCSS Module Örneği

```scss
.wrapper {
  padding: $desktop-padding;
  border-radius: $desktop-border-radius;
  background-color: $card-bg;

  @include for-tablet {
    padding: $tablet-padding;
    border-radius: $tablet-border-radius;
  }

  @include for-mobile {
    padding: $mobile-padding;
    border-radius: $mobile-border-radius;
  }

  &--active {
    border-color: $primary;
  }

  &__title {
    font-size: 18px;
    color: $text-primary;
  }
}
```

### UI Varyantları ve Boyutları
- Enum tabanlı variant'lar: `PRIMARY`, `DARK`, `LIGHT`, vb.
- Enum tabanlı boyutlar: `SMALL`, `MEDIUM`, `LARGE`
- clsx ile conditional class uygulaması yapılır.

## 9. API ve Veri Çekme

### Servis Yapısı
- Her API endpoint grubu için ayrı bir servis dosyası: `/services/modules/domain.service.ts`
- Axios client instance'ları: `/services/clients/` (farklı base URL'ler için ayrı client)
- Request/Response interceptor'lar: `/services/interceptors/`

### Servis Dosyası Örneği

```tsx
import apiClient from '@/services/clients/api-client';

export default {
  search: async (params: SearchParams) =>
    apiClient.get('/search', { params }),

  sendMessage: async (data: SendMessageData) =>
    apiClient.post('/messages', data),
};
```

### Veri Çekme Yaklaşımları
1. **Client-side:** Hook'lar + servis çağrıları ile yapılır.
2. **Real-time:** WebSocket entegrasyonu (`useSocket` hook) ile canlı veri akışı sağlanır.

### Ortam Yapılandırması
- Vite ortam değişkenleri: `import.meta.env.VITE_*`
- `.env`, `.env.development`, `.env.production` dosyaları kullanılır.

## 12. Genel Kurallar

1. **Relative import kullanma**, her zaman `@/` path alias'ı kullan.
2. **Component'leri her zaman `memo()` ile sar.**
3. **Props için `type` kullan**, `interface` değil.
4. **Her component kendi klasöründe olsun**: `ComponentName/index.tsx` + `index.module.scss`
5. **Servisler object olarak default export edilir**, fonksiyonlar named export.
6. **SCSS değişkenleri global olarak tanımlanır**, component'lerde import gerekmez.
7. **BEM naming convention** kullan CSS class'larında.
8. **Enum'lar ayrı dosyalarda tanımlanır**, `UPPER_SNAKE_CASE` değerler ile.
9. **Hook'lar `use` prefix'i ile adlandırılır.**
10. **Tek dil (Türkçe)** - i18n kütüphanesi kullanılmaz.
